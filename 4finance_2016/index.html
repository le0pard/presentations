Depression

How developers feel

How developers feel

Example: React.js

  1. Facebook has a problem — so many state changing events that it is hard to keep track what’s going on
  2. Facebook promotes new paradigm with buzzwords: functional, virtual DOM, components
  3. Mania: Facebook has created the front-end framework of the future! Let’s write everything in react from now on!
  4. Wait there is a lot of work, but no quick return on investment!
  5. React is great for advanced one page app with lots of real-time notifications, but does not necessarily pay off for simpler applications

Example: TDD is dead by DHH

  1. David Heinemeier Hansson realises that it is hard to do TDD with Rails as this framework doesn’t have architecture supporting good OOP
  2. Hype starts with DHH blog post and conference talk. Hype keywords: TDD is DEAD
  3. Let’s skip tests! Our Guru says so. We didn’t write them anyway
  4. Wait! Even fewer things work now than before. We’ve built a buggy code
  5. “TDD is not dead or alive. TDD is subject to tradeoffs, including risk of API changes, skill of practitioner and existing design” — Kent Beck

Example: Microservices

  1. Big monolith application scales hard. There is a point when we can break them down into services
  2. Hype keywords: scalability, loose coupling, monolith
  3. Let’s rewrite all to services! We have a ‘spaghetti code’ because we have a monolith architecture! We need to rewrite everything to microservices!
  4. Shit! It is now way slower to develop the app, difficult to deploy and we spend a lot of time tracking bugs across multiple systems
  5. Microservices require a lot of devops skills in the team and with right investment might pay off as a way to scale the system and team
