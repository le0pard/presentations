React Native

  • React!!!
  • It is not hybrid/html5 app
  • Use native code when you need to
  • Fast view changes - no need to recompile

Example

import React from 'react';
import {Text, View} from 'react-native';

class RubyMeditationMobile extends React.Component {
  render() {
    return (
      <View>
        <Text>Hello Rubymeditation!</Text>
      </View>
    );
  }
}

Example

import {AppRegistry} from 'react-native';

AppRegistry.registerComponent(
  'RubyMeditationMobile',
  () => RubyMeditationMobile
);

Example

$ react-native start // start dev server

$ react-native run-ios // run app in ios emulator

$ react-native run-android // run app in android emulator

React Native Benefits

  • You are using react way to build apps (components, react infrastructure)
  • You can share codebase with your react web application: actions, action types, reducers, API functions (REST, GraphQL)
  • Fast development for small team (one team for iOS and Android platform) and you can see results immediately
  • Very good fit for apps, like: conferences app, booking app, catalogs app, shopping app, etc

React Native Disadvantages

  • React Native provides only basic components and APIs
  • Components built on top of the React native components are more flexible, but they are decreasing performance for an interface
  • In case you need some special (native) component you need to have knowledge in Objective-C/Swift + Java/Kotlin
  • You have only one JS thread for interface, so heavy animations or some heavy JS logic will block interface
  • Still is not very stable in development process

React Native Disadvantages

  • iOS first, Android second: support of the Android is not full and not perfect (shadows, image placeholders, etc)
  • Top level debug is simple, but if you need go deeper - you need to read native code
  • Outdated/not complete documentation
  • BSD + Patents license (not critical)
  • Not fit for apps like: games, low-level apps (that get some system info)
